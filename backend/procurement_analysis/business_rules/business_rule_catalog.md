# Legacy Business Rule Catalog (LPIE)

> **Facts only.** Each rule is a behaviour **observed in the legacy system**, traceable to evidence.
> Do **not** invent or "improve" rules here — this catalog records what the legacy app *does*. (The
> redesigned, forward-looking rules live separately in `docs/Procurement_Business_Rules.md`; this LPIE
> catalog is the *as-is* legacy record that informs it.)

## Rule ID scheme

`LR-<CATEGORY>-NNN` (Legacy Rule). Categories below. IDs are permanent, never reused.

| Category code | Category |
|---------------|----------|
| STK | Stock Rules |
| SUP | Supplier Rules |
| PUR | Purchase Rules |
| SAL | Sales Rules |
| PEN | Pending Rules |
| EXP | Expiry Rules |
| BAT | Batch Rules |
| ORD | Order Rules |
| APR | Approval Rules |
| EXPT | Export Rules |
| SEC | Security Rules |

## Per-rule fields (use `_TEMPLATE_rule.md`)

Rule ID · Title · Description · Trigger · Conditions · Calculation · Validation · Affected Tables ·
Affected Screens · Priority · Evidence Source · Status.
